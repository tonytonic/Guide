
## 58. Lot 52 — session record de récidives exactes confirmées

10 pages agrémentées. **4 erreurs corrigées**, dont **trois récidives
exactes** d'erreurs déjà identifiées et documentées lors de sessions
précédentes — toutes détectées dès la première lecture des tableaux IDCC,
avant même toute recherche complémentaire.

**controleur-sncf-conducteur-train-hs** — page distincte de controleur-
sncf-heures-supplementaires (lot 32), affichait le **même IDCC 2002**
(blanchisserie/pressing) déjà identifié comme faux pour ce secteur
ferroviaire. Corrigée vers **IDCC 3217** (transport ferroviaire), avec
nuance sur le statut spécifique des agents SNCF statutaires (hors système
IDCC classique).

**architecture-interieure-decoration-heures-supplementaires** — page
distincte d'architecture-heures-supplementaires (lot 39), affichait le
**même IDCC 2609** (ETAM bâtiment) déjà identifié comme faux pour
l'architecture. Corrigée vers **IDCC 2332** (entreprises d'architecture),
déjà confirmé à deux reprises (architecte-salarie lot 37, architecture
lot 39) — troisième page du site sur ce métier désormais alignée.

**agent-quai-manutentionnaire-hs** — affichait le **même IDCC 1611**
(logistique de communication écrite directe) déjà identifié comme faux
pour magasinier-cariste-hs au lot 36. Cette fois, une recherche
complémentaire a permis d'affiner la réponse : en port maritime, la
convention **ports et manutention (IDCC 3017)** s'applique explicitement
aux dockers ; hors port (entrepôt logistique classique), aucune convention
dédiée n'existe et le secteur réel de l'employeur détermine le code
(transport routier IDCC 16 notamment). Page corrigée avec cette double
nuance plutôt que le chiffre non vérifiable.

**informatique-heures-supplementaires** — affichait **IDCC 1678**,
introuvable dans toutes les sources consultées malgré plusieurs
recherches ciblées (probable confusion de chiffres avec 1679, inspection
d'assurance). Corrigé vers **IDCC 1486** (Syntec), le code standard et
très largement documenté pour les entreprises de services numériques
(ESN, ex-SSII), déjà confirmé à de nombreuses reprises ce chantier.

**Point méthodologique** : ce lot confirme que certaines erreurs se
propagent de façon quasi systématique à toutes les pages d'un même
secteur générées lors du même processus initial — trois des quatre
erreurs de ce lot reproduisaient exactement des erreurs déjà cataloguées
sur des pages sœurs. Un grep ciblé sur les codes déjà identifiés comme
faux (2002, 2609, 1611, 200, 1767, 2583, 1659, 1147, 1996...) à travers
l'ensemble du corpus restant pourrait permettre de repérer plus
efficacement les prochaines occurrences, plutôt que d'attendre qu'elles
apparaissent lot après lot.
